## 3. Conventions
- Design tokens: CSS custom properties on `:root` in app/globals.css — see
  docs/DESIGN_SYSTEM.md §4.1–4.5 for full table (colours, spacing, radius, shadow).
- Reference tokens via Tailwind arbitrary values: `bg-[var(--c-surface)]`,
  `rounded-[var(--radius-lg)]`, `shadow-[var(--shadow-card)]`.
- Font utilities: `.font-display` (Space Grotesk), `.font-data` (JetBrains Mono).
  Body defaults to DM Sans via `--font-body` — no utility needed.
- Weight ceiling: Space Grotesk ≤600, DM Sans ≤500. Never exceed.
- Letter-spacing scale: −0.8px at 30px headline → normal (0) below 16px. Mono ALL CAPS
  labels get +0.06–0.14em (wider, not tighter).
- Shape rule: section cards/inputs/buttons = 4–6px radius (sharp). Chips/badges/pills =
  `--radius-pill` (9999px). Never blur this line.
- Components: PascalCase in /components, one component per file, `'use client'` only
  where state/effects/handlers are used.
